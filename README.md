# CrackedMilk Internal Repository

Welcome to the internal repository for **CrackedMilk**.

This repository serves as the central hub for our internal marketing strategies, operational workflows, and the source code for our public-facing landing page.

## Project Structure

The repository is organized into two main areas perfectly separating our internal documentation from the user-facing web logic:

### 1. Internal Documentation (`docs/`)
This directory contains all confidential strategy and operational documents:
- [Marketing Strategy](docs/strategy.md): Outlines our high-level marketing approach and brand positioning.
- [AI Video Workflow](docs/ai_video_workflow.md): Standard operating procedures for AI video generation and rendering.
- [Onboarding Framework](docs/onboarding_framework.md): Guide and framework for onboarding new team members and creators.
- [Implementation Options](docs/implementation_options.md): Technical options and tradeoffs for systems we use.

### 2. Public Website (`landing-page/`)
This directory solely contains the source code for the public-facing CrackedMilk website. It does not contain any internal documentation.
- **Run Locally:** `cd landing-page/` then run `npx serve .`

***

*Note: This repository is private and the documentation within should not be deployed with the web page.*
